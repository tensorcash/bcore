# C++ Golden Vector Integration

## Overview

The C++ test suite supports testing with real BLS12-381 Groth16 proofs generated by gnark, instead of mock data.

## Files

### 1. Golden Vector Loader (`src/test/util/golden_vector_loader.h/cpp`)

Provides utilities to load golden vectors from JSON:

```cpp
namespace golden_vectors {
    struct GoldenVector {
        std::string name;
        Witness witness;
        std::vector<unsigned char> proof_bytes;      // 192 bytes
        std::vector<unsigned char> public_inputs_bytes;  // 128 bytes
        std::vector<unsigned char> vk_bytes;         // ~872 bytes
        bool should_fail;

        // Helpers
        uint256 GetChainSeparator() const;
        uint256 GetAssetId() const;
        uint32_t GetRootHeight() const;
        uint256 GetTfrAnchor() const;
    };

    std::optional<GoldenVector> LoadGoldenVector(const std::string& vector_name);
    std::vector<GoldenVector> LoadAllGoldenVectors();
    bool GoldenVectorsAvailable();
}
```

Features:
- Locates the golden vectors JSON produced by the kyc-prover vector generator
- Parses JSON using UniValue
- Extracts proof, public inputs, and VK from hex strings
- Provides helper methods to extract fields from public inputs

### 2. Golden Vector Tests (`src/test/groth16_golden_tests.cpp`)

Test suite using real proofs:

```cpp
BOOST_AUTO_TEST_SUITE(groth16_golden_tests)

// Test cases:
BOOST_AUTO_TEST_CASE(groth16_golden_valid_proof)
BOOST_AUTO_TEST_CASE(groth16_golden_corrupted_proof)
BOOST_AUTO_TEST_CASE(groth16_golden_wrong_public_inputs)
BOOST_AUTO_TEST_CASE(groth16_golden_cross_vk_rejection)
BOOST_AUTO_TEST_CASE(groth16_golden_root_age_enforcement)
BOOST_AUTO_TEST_CASE(groth16_golden_tfr_anchor_enforcement)
BOOST_AUTO_TEST_CASE(groth16_golden_invalid_vectors_no_proof)

BOOST_AUTO_TEST_SUITE_END()
```

Key features:
- Conditionally skips if vectors are not available
- Tests valid proof verification (should succeed)
- Tests corrupted proof rejection
- Tests wrong public inputs (pairing failure)
- Tests policy enforcement (root age, anchor)
- Validates invalid vectors have no proofs

## Usage

### Generating Golden Vectors

Golden vectors are produced by the kyc-prover vector generator (`gentest`), which lives in a separate repository:

```bash
# Build gentest
go build -o gentest ./cmd/gentest

# Generate vectors (takes ~60 seconds)
mkdir -p vectors
./gentest -output vectors
```

Output:
- `vectors/proving_key.bin` - Groth16 proving key (~100MB)
- `vectors/verification_key.bin` - Groth16 verification key (~2KB)
- `vectors/golden_vectors.json` - Test vectors with proofs

### Running C++ Tests

```bash
# Build with tests enabled
cmake -B build -DBUILD_TESTS=ON
cmake --build build -j$(nproc)

# Run all groth16 tests
./build/src/test_bitcoin --run_test=groth16_golden_tests

# Run specific test
./build/src/test_bitcoin --run_test=groth16_golden_tests/groth16_golden_valid_proof

# Show verbose output
./build/src/test_bitcoin --run_test=groth16_golden_tests --log_level=test_suite
```

## Test Cases Explained

### 1. Valid Proof Verification

```cpp
auto golden = LoadGoldenVector("valid");
auto result = VerifyGroth16WithPolicy(golden.proof_bytes, golden.public_inputs_bytes, golden.vk_bytes, ctx);
BOOST_CHECK(result == VerifyError::OK);
```

This verifies that real BLS12-381 Groth16 proofs from the circuit verify in C++.

### 2. Corrupted Proof

```cpp
golden.proof_bytes[10] ^= 0xFF;  // Flip a byte
auto result = VerifyGroth16WithPolicy(...);
BOOST_CHECK(result != VerifyError::OK);
```

Ensures that even single-byte corruption is detected.

### 3. Wrong Public Inputs

```cpp
golden.public_inputs_bytes[40] ^= 0xFF;  // Mutate asset_id
auto result = VerifyGroth16WithPolicy(...);
BOOST_CHECK(result == VerifyError::PairingFailed);
```

Tests that proof fails if public inputs don't match.

### 4. Root Age Enforcement

```cpp
uint32_t root_height = golden.GetRootHeight();
VerificationContext ctx{
    .max_root_age = 100,
    .current_height = root_height + 200  // Too old
};
auto result = VerifyGroth16WithPolicy(...);
BOOST_CHECK(result == VerifyError::RootTooOld);
```

Validates that compliance root age is checked correctly.

### 5. TFR Anchor Enforcement

```cpp
uint256 correct_anchor = golden.GetTfrAnchor();
uint256 wrong_anchor = uint256S("0x1111...");

VerificationContext ctx{
    .anchor_commitment = MakeUCharSpan(wrong_anchor)
};
auto result = VerifyGroth16WithPolicy(...);
BOOST_CHECK(result == VerifyError::AnchorMismatch);
```

Tests anchor commitment validation.

### 6. Invalid Vectors Have No Proofs

```cpp
for (auto& name : {"invalid_secret", "invalid_age", "invalid_country", "invalid_merkle"}) {
    auto vec = LoadGoldenVector(name);
    BOOST_CHECK(vec.should_fail);
    BOOST_CHECK(vec.proof_bytes.empty());
}
```

Confirms that witnesses violating circuit constraints cannot produce proofs.

## Comparison: Mock Tests vs Golden Tests

### Mock Tests (`groth16_tests.cpp`)

Pros:
- No external dependencies
- Fast execution (<1ms per test)
- Test error paths and edge cases
- Always available

Cons:
- Don't test real cryptographic verification
- Use fake curve points that fail pairing
- Can't catch bugs in actual proof verification

Use cases:
- Format validation
- Error code paths
- Policy enforcement logic
- Size checks

### Golden Tests (`groth16_golden_tests.cpp`)

Pros:
- Test real cryptographic verification
- Validate gnark circuit integration
- Catch subtle pairing/serialization bugs
- Exercise consensus code with real proofs

Cons:
- Require golden vectors to be generated
- Slower (~100ms per test due to pairing)
- Need ~100MB of disk space for proving key

Use cases:
- End-to-end verification
- Regression testing after crypto changes

## Continuous Integration

Golden vectors can be generated in CI by the kyc-prover vector generator and published as a build artifact, then consumed by the C++ test job, which downloads the artifact and runs `test_bitcoin --run_test=groth16_golden_tests` against it.

## Debugging Tips

### Vector Not Found

If the suite reports that golden vectors were not found, regenerate them with the kyc-prover vector generator (`./gentest -output vectors`) and ensure the output is placed where the loader expects it.

### Test Fails with PairingFailed

```
Error: Golden vector 'valid' should verify successfully. Error: 6
```

Possible causes:
1. VK mismatch (regenerated with different setup)
2. Proof serialization bug
3. Public inputs format mismatch
4. BLST version incompatibility

Debug:
```cpp
// Add to test
std::cout << "Proof size: " << golden.proof_bytes.size() << std::endl;
std::cout << "Inputs size: " << golden.public_inputs_bytes.size() << std::endl;
std::cout << "VK size: " << golden.vk_bytes.size() << std::endl;
std::cout << "Root height: " << golden.GetRootHeight() << std::endl;
```

### Vectors Out of Date

If the circuit is modified, regenerate the vectors with the kyc-prover vector generator:

```bash
rm -rf vectors/
./gentest -output vectors
```

## Summary

The golden vector integration enables C++ unit tests to:

1. Verify real BLS12-381 Groth16 proofs
2. Test consensus validation with actual gnark proofs
3. Catch cryptographic bugs
4. Validate policy enforcement on real data

Combined with mock tests, this provides comprehensive coverage:
- Mock tests: fast, always-available error path testing
- Golden tests: cryptographic correctness validation
